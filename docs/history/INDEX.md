# History — frozen, read-only

This tree is **archaeology**. It preserves the reasoning path that produced Arius's current design, in the formats the project used before settling on ADRs + design docs. **Nothing here is maintained.** It is kept because the narratives hold *intent* (rejected alternatives, the "why") that the living docs summarise but do not reproduce in full.

If you want the current truth, start at [`../README.md`](../README.md) (the doc map), [`../design/`](../design/README.md), and [`../decisions/`](../decisions/README.md). Come here only to understand *why* a decision was made the way it was.

## How the documentation evolved

Arius's docs grew through four overlapping eras (dates from git history):

| Era | Window | System | Where its durable intent lives now |
|---|---|---|---|
| 1 | 2026-03-10 → 2026-06-08 | **OpenSpec** — capability specs + per-change proposal/design/tasks | Mechanical specs collapsed into [`../design/`](../design/README.md); rationale lifted into ADRs [0013](../decisions/adr-0013-core-host-separation.md), [0014](../decisions/adr-0014-encryption-format-and-recoverability.md), [0015](../decisions/adr-0015-chunk-index-scalability.md), [0016](../decisions/adr-0016-multi-machine-cache-coherence.md), [0017](../decisions/adr-0017-idempotent-non-distributed-recovery.md). Change narratives frozen in [`openspec-archive/`](openspec-archive/2026-03-24-arius-core-foundation/design.md). |
| 2 | 2026-04-19 → 2026-05-11 | **Superpowers** — RFC-style design docs | Superseded by ADRs [0001](../decisions/adr-0001-structure-representative-e2e-coverage.md), [0006](../decisions/adr-0006-build-filetrees-from-hashed-directory-staging.md), [0008](../decisions/adr-0008-introduce-internal-filesystem-domain-types.md), [0009](../decisions/adr-0009-clarify-test-fixture-boundaries.md). Frozen in [`superpowers/`](superpowers/2026-04-20-shared-test-infrastructure-design.md). |
| 3 | 2026-04-24 → **today** | **ADRs (MADR)** | **Live** — this is the current decision home: [`../decisions/`](../decisions/README.md). |
| 4 | 2026-06-10 → **today** | **Full agentic** — saved plans + conversations | Implemented in code; intent folded into ADRs and [`../design/`](../design/README.md). Plans frozen in [`agentic-plans/`](agentic-plans/2026-06-15-ztd/PLAN.md). |

The decisive moment was 2026-04-24 (`feat: replace superpower specs & plans with MADR`): the superpowers *plans* were deleted and ADRs became the decision record. OpenSpec persisted alongside ADRs until 2026-06-08, after which the project worked "full agentic" and saved its plans rather than maintaining a spec system.

## What's here

- [`openspec-archive/`](openspec-archive/2026-03-24-arius-core-foundation/design.md) — the `proposal.md` + `design.md` of each completed OpenSpec change (the `tasks.md` checklists and Gherkin spec deltas were dropped as non-durable). The `design.md` files carry the richest rejected-alternatives reasoning — e.g. the foundational `2026-03-24-arius-core-foundation`, `2026-03-31-aes256gcm-encryption`, and the chunk-index scalability arc (`2026-05-29`, `2026-06-02`, `2026-06-08`).
- [`superpowers/`](superpowers/2026-04-20-shared-test-infrastructure-design.md) — the four superpowers design docs (shared test infrastructure, the representative E2E workflow, filetree staging/build, the core path-typing sweep).
- [`agentic-plans/`](agentic-plans/2026-06-15-ztd/PLAN.md) — `PLAN`/`CONVO`/report markdown saved during the agentic era (raw `.txt` conversation transcripts and UI mockup assets were dropped). Includes the dynamic-shard-length and chunk-index scalability plans that fed [ADR-0015](../decisions/adr-0015-chunk-index-scalability.md), and the zstd evaluation that fed [ADR-0012](../decisions/adr-0012-zstd-as-new-compression-algorithm.md). The two latest were lifted out of `openspec/changes/` (a leftover habit — OpenSpec ended 2026-06-08): the [v5→v7 migration tool](agentic-plans/2026-06-16-migration-tool/PLAN.md) plan — its durable intent now lives in the [migration](../design/migration.md) design doc, plus the legacy AES-256-CBC read back-compat (PBKDF2 = 10,000) documented in [encryption](../design/core/shared/encryption.md) — and the [chunk-index repair/flush unification](agentic-plans/2026-06-19-chunkindex-repair-tweaks/CONVO1.md) conversation, folded into [chunk-index](../design/core/shared/chunk-index.md) and [repair-chunk-index](../design/core/features/repair-chunk-index.md). The follow-up [migration sidecar + historical states](agentic-plans/2026-06-21-migration-sidecar/CONVO.txt) conversation (the archive-tier chunk metadata sidecar and the v3/v5 historical-snapshot replay) landed in [ADR-0018](../decisions/adr-0018-archive-tier-metadata-sidecar.md) and the [migration](../design/migration.md) design doc. The [fix-snapshot-totalsize](agentic-plans/2026-06-22-fix-snapshot-totalsize/2026-06-22-152052-when-running-an-archive-command-on-an-existing-a.md) investigation (snapshot size metrics, then making the web Statistics figures complete and fast) and the [Arius.Web tweaks](agentic-plans/2026-06-18-ariusweb-tweaks/CONVO.md) conversation (the snapshot bar promotion, Statistics redesign, Properties drawer with passphrase rotation + delete, and the Karma→Playwright move) landed in the [chunk-index](../design/core/shared/chunk-index.md) / [read-queries](../design/core/features/queries.md) / [web host](../design/hosts/web.md) design docs and the [web-ui guide](../guide/web-ui.md). The [web per-repo logging](agentic-plans/2026-06-23-web-per-repo-logging/PLAN.md) plan (routing Web→Core operations to a shared per-repository rolling log file in the same `~/.arius/.../logs/` location the CLI writes) landed in the [logging](../design/cross-cutting/logging.md#per-host-setup) and [web host](../design/hosts/web.md) design docs and the [deployment guide](../guide/deployment.md). The [file/folder filter](agentic-plans/2026-06-23-file-folder-filter/PLAN.md) plan (configurable archive exclusions, defined centrally in Arius.Core to avoid host drift) landed in [ADR-0019](../decisions/adr-0019-central-file-exclusion-configuration.md) and the [archive-command](../design/core/features/archive-command.md) design doc. The [storage accounts & provider-agnostic pricing](agentic-plans/2026-06-27-storage-accounts-and-provider-agnostic-pricing/README.md) plan (storage-account region management + edit flyout, a region-aware cost-by-tier view, real Azure Retail Prices rates with Cool/Cold retrieval + egress in the restore estimate, and moving Azure pricing into Arius.AzureBlob behind the `IStorageCostEstimator` port) landed in [ADR-0020](../decisions/adr-0020-provider-agnostic-cost-estimation.md), the [cost estimation](../design/core/shared/cost.md) design doc, the [restore-command](../design/core/features/restore-command.md) / [read-queries](../design/core/features/queries.md) / [web host](../design/hosts/web.md) design docs, and the [web-ui guide](../guide/web-ui.md).
