  What changed

  ListQueryHandler (src/Arius.Core/Features/ListQuery/ListQueryHandler.cs)
  - Handle now creates a bounded Channel<RepositoryEntry> (capacity 4096, Wait
  full-mode for backpressure). A producer task walks the tree and writes
  entries; the consumer side yields from ReadAllAsync. Producer failures surface
  through Writer.Complete(ex), and abandoning enumeration early cancels the
  producer via a linked CTS in the finally, so nothing leaks.
  - The recursive async-iterator walk is replaced with an iterative depth-first
  walk using an explicit Stack<RecursionTarget>. Listing order is unchanged
  (pre-order: directories, then files, then descend), but emitting an entry is
  now O(1) instead of O(depth) through nested iterator chains.
  - Per-directory file processing no longer materializes the whole directory:
  cloud files are processed in batches of 512, each batch doing one chunk-index
  size lookup before its entries are written to the channel. The old
  visibleCloudFiles/hashes/yieldedFileNames full-directory lists are gone ‚ÄĒ a
  cloud file consumes its local counterpart by removing it from the local-files
  dictionary, and whatever remains afterwards is emitted as local-only.
  - Fixed a latent unbounded-memory bug: BuildLocalDirectorySnapshot enumerated
  local files with SearchOption.AllDirectories, pulling the entire local subtree
  into a dictionary at every recursion level (O(n¬∑depth) work, and nested
  local-only files could be emitted twice ‚ÄĒ once at the ancestor with a full
  path, once in their own directory). It now enumerates TopDirectoryOnly,
  matching the documented "immediate children" semantics; recursion handles the
  rest.
  - Also made consumer-side cancellation prompt: ReadAllAsync only observes the
  token while waiting, so I check it per item ‚ÄĒ this showed up as a flaky
  cancellation test before the fix, and the suite is now stable across repeated
  runs.

  What stays in memory at any moment: one directory's tree entries (inherent to
  the IFileTreeService.ReadAsync API), one 512-entry lookup batch, the local
  snapshot of one directory, the DFS stack of unvisited siblings, and at most
  4096 buffered channel entries.

  LsVerb (src/Arius.Cli/Commands/Ls/LsVerb.cs)
  - The Spectre Table, which accumulated every row before rendering, is replaced
  with a fixed-width header plus one line written per file as entries arrive ‚ÄĒ
  first results appear immediately and memory stays flat.
  - The AnsiConsole recorder is removed: it captured the entire console output
  in memory for the audit log, which would defeat the bounded-memory goal at
  millions of rows. The audit log now gets a ls completed: N file(s) summary
  instead, and the start parameters are still logged by the handler.

  The Explorer view model consumes the same mediator stream, so it picks up the
  streaming/prefetching behavior with no changes.
